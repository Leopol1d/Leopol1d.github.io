    • 强调通过Nginx反向代理实现负载均衡和高可用，以提高系统的稳定性和性能。

RESTFUL 标准

整个分类管理都是严格使用 RESTFUL 标准来做的，我们来总结一下。

  1. 资源名称用复数，而非单数。

即使用 /categories 而不是用 /category

  1. CRUD 分别对应：

增加： post 删除： delete 修改： put 查询： get

  1. id 参数的传递都用 /id方式。

如编辑和修改： /categories/123

  1. 其他参数采用?name=value的形式

如分页参数 /categories?start=5

  1. 返回数据

查询多个返回 json 数组 增加，查询一个，修改 都返回当前 json 数组 删除 返回空

遇到的困难

另外还提供了一个奇怪的方法，removeOrderFromOrderItem，它的作用是把订单里的订单项的订单属性设置为空。。。 听上去绕口吧。 再用代码说一下，比如有个 order, 拿到它的 orderItems， 然后再把这些orderItems的order属性，设置为空。 为什么要做这个事情呢？ 因为SpringMVC ( springboot 里内置的mvc框架是 这个东西)的 RESTFUL 注解，在把一个Order转换为json的同时，会把其对应的 orderItems 转换为 json数组， 而 orderItem对象上有 order属性， 这个order 属性又会被转换为 json对象，然后这个 order 下又有 orderItems 。。。 就这样就会产生无穷递归，系统就会报错了。 所以这里采用 removeOrderFromOrderItem 把 OrderItem的order设置为空就可以了。 那么为什么不用 @JsonIgnoreProperties 来标记这个字段呢？ 因为后续我们要整合Redis，如果标记成了 @JsonIgnoreProperties 会在和 Redis 整合的时候有 Bug, 所以还是采用这种方式比较好。 这些都是站长掉进去的坑~

代码比较复制代码

增加两个 removeCategoryFromProduct 方法。 这个方法的用处是删除Product对象上的 分类。 为什么要删除呢？ 因为在对分类做序列还转换为 json 的时候，会遍历里面的 products, 然后遍历出来的产品上，又会有分类，接着就开始子子孙孙无穷溃矣地遍历了，就搞死个人了 而在这里去掉，就没事了。 只要在前端业务上，没有通过产品获取分类的业务，去掉也没有关系。

在Java项目中，Redis、Elasticsearch和Shiro分别具有不同的作用：

  1. Redis：

    Redis是一个内存数据库，通常用作缓存服务器。在Java项目中，Redis的作用可以包括：

    • 缓存数据： Redis可以存储频繁访问的数据，例如数据库查询结果、用户会话信息等。这有助于提高系统性能，减轻数据库的负担。

    • 消息队列： Redis支持发布/订阅模式，可以用作消息队列，用于异步处理任务、事件通知等。

    • 分布式锁： Redis可以用于实现分布式锁，确保在多个应用实例之间对共享资源的安全访问。

    • 计数器和统计： 可以用Redis来维护计数器，记录应用中的各种统计数据。

  2. Elasticsearch：

    Elasticsearch是一个开源搜索和分析引擎，用于处理大规模的文本数据。在Java项目中，Elasticsearch的作用包括：

    • 全文搜索： Elasticsearch可以用于实现高效的全文搜索功能，例如搜索引擎、文档检索等。

    • 日志和事件分析： 用于存储和分析大量日志和事件数据，以便进行监视和报告。

    • 数据分析和可视化： 可以将Elasticsearch与Kibana等工具结合使用，实现数据分析和可视化。

    • 实时数据处理： 支持实时索引和查询，适用于需要及时反馈的应用。

  3. Shiro：

    Apache Shiro是一个强大且易于使用的身份验证和访问控制框架。在Java项目中，Shiro的作用包括：

    • 身份验证： Shiro可以处理用户身份验证，包括用户名和密码验证、单点登录（SSO）等。

    • 授权： 可以配置Shiro来实现基于角色和权限的访问控制，确保只有授权用户可以访问受保护的资源。

    • 会话管理： Shiro支持会话管理，可以处理用户会话的创建、维护和过期等问题。

    • 加密和哈希： 提供了密码加密和哈希的工具，用于存储用户密码的安全性。

这些技术通常在Java项目中用于不同的方面，但它们可以协同工作以增强项目的性能、安全性和功能。例如，您可以使用Redis作为Shiro会话管理的缓存存储，以提高性能。同时，您还可以将Elasticsearch用于搜索和分析项目的日志数据。这些技术的选择和使用取决于项目的具体需求和目标。

JPA（Java Persistence API）是Java平台上用于管理持久化数据的一种规范，它提供了一组API和一种标准的方式来访问和管理数据库中的数据。JPA的目标是简化Java应用程序与数据库之间的交互，减少了开发者需要编写的数据库访问代码，从而提高了开发效率和可维护性。

以下是关于JPA的一些关键概念和特点：

  1. 对象关系映射（ORM）： JPA允许开发者将Java应用程序中的Java对象与数据库中的表进行映射。这意味着您可以使用面向对象的方式来操作数据库，而不必编写复杂的SQL查询。

  2. 实体类： 在JPA中，数据库表的每一行数据都由一个Java类表示，这个Java类称为实体类。实体类通常使用注解来描述如何与数据库表进行映射。

  3. EntityManager： EntityManager是JPA的核心接口之一，它用于管理实体对象的生命周期、执行CRUD（创建、读取、更新、删除）操作以及执行查询操作。

  4. JPQL（Java Persistence Query Language）： JPQL是一种与数据库无关的查询语言，类似于SQL，但是使用实体类和属性名而不是数据库表和列名来进行查询。JPQL查询是类型安全的，因此编译时可以捕获到查询错误。

  5. 关联关系： JPA支持实体之间的关联关系，包括一对一、一对多、多对一和多对多关系。这使得在实体之间建立关系和导航数据变得非常容易。

  6. 事务管理： JPA提供了事务管理的支持，确保在数据库操作期间的数据一致性和完整性。

  7. 兼容性： JPA是一个Java EE（Enterprise Edition）规范，但也可以独立于Java EE在Java SE（Standard Edition）应用程序中使用。

  8. 提供商实现： JPA规范由多个JPA提供商实现，包括Hibernate、EclipseLink、OpenJPA等。开发者可以根据项目需求选择合适的提供商。

总的来说，JPA是一种强大的持久化技术，可以使Java开发者更轻松地操作数据库，减少了与数据库交互的复杂性，提高了代码的可维护性。它广泛用于Java应用程序的开发，特别是在企业级应用中，以便于管理和维护数据。
